## 用户需求

创建一个Go项目，实现数据源管理功能：

1. 前端页面用于输入数据库IP、端口、用户名、密码、数据源类型，提供两个按钮：测试连通性和保存数据源
2. 后端提供2个API接口实现对应功能，数据存储到SQLite
3. 后端支持MySQL、Oracle、PostgreSQL、Redis、SQL Server、SQLite等多种数据源，每种数据源实现不同的连通性测试逻辑

## 核心功能

- 数据源表单：IP、端口、用户名、密码、数据源类型选择
- 测试连通性：验证数据库连接是否可用，返回连接结果
- 保存数据源：将连接信息持久化到SQLite数据库
- 多数据源支持：MySQL、PostgreSQL、Oracle、Redis、SQL Server、SQLite

## 技术栈选择

- 后端：Go + Gin框架
- 前端：纯HTML + JavaScript (原生Fetch API)
- 存储：SQLite (用于存储数据源配置)

## 关键技术实现

### 1. 连通性测试驱动

- MySQL: go-sql-driver/mysql
- PostgreSQL: lib/pq
- Oracle: sijms/go-ora/v2
- Redis: go-redis/redis/v8
- SQL Server: denisenkom/go-mssqldb
- SQLite: mattn/go-sqlite3

### 2. API设计

- `POST /api/datasource/test` - 测试连通性，接收数据源信息，返回连接是否成功
- `POST /api/datasource/save` - 保存数据源，将配置存入SQLite，返回保存结果

### 3. SQLite表结构

```sql
CREATE TABLE datasources (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    db_type TEXT NOT NULL,
    host TEXT NOT NULL,
    port INTEGER NOT NULL,
    username TEXT,
    password TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 页面设计

采用现代简洁的设计风格，使用卡片式布局和渐变配色。

### 页面结构（共4个功能块）

1. **头部区域**：页面标题"数据源管理"，带有渐变背景
2. **表单卡片区域**：数据源配置表单，包含类型选择器、输入框组
3. **操作按钮区域**：两个并排的按钮（测试连通性、保存数据源）
4. **结果提示区域**：显示操作结果（成功/失败提示）

### 设计风格

- 简洁现代风，配合微阴影和圆角
- 渐变头部背景
- 清晰的表单布局
- 醒目的操作按钮
